- 交付一个可直接打开运行的网页大屏，**无需后端、无需构建工具、无需联网也能看到骨架**（图表库可从 CDN 加载，加载失败时要有兜底）。

- 数据全部为**前端 Mock**，并通过定时器模拟「实时更新」，让画面持续动起来。

## 二、设计规范（这是“漂亮”的关键，必须遵循）
1. **设计基准分辨率 1920 × 1080**：整体用绝对定位栅格；通过「等比缩放适配」让任意屏幕铺满且不变形：
   - 外层容器 `#screen` 固定宽高 `1920 × 1080`；
   - 监听 `resize`，计算 `scale = Math.min(innerWidth / 1920, innerHeight / 1080)`，对 `#screen` 施加 `transform: scale(scale)` 并居中。
   - 关键代码：
     ```js
     const charts = [];
     function fitScreen() {
       const scale = Math.min(innerWidth / 1920, innerHeight / 1080);
       document.getElementById('screen').style.transform = `scale(${scale})`;
     }
     addEventListener('resize', () => { fitScreen(); charts.forEach(c => c.resize()); });
     fitScreen();
     ```
2. **视觉主题**：深色科技风。背景深蓝渐变（如 `#060d2a → #0a1a44`）叠加细微网格/星点；主色 cyan `#22d3ee`、蓝 `#3b82f6`、绿 `#34d399`、橙 `#fbbf24`、紫 `#a78bfa`。
3. **装饰元素**：面板用「切角 / 发光边框」科技风（CSS 伪元素 + `linear-gradient` 描边与四角装饰）；标题栏左右对称流光线条与中心高光。
4. **字体**：中文无衬线（`"Microsoft YaHei","PingFang SC",sans-serif`）；数字/英文用等宽或 DIN 风格体现数据感；主标题 32–40px，面板标题 18–20px。
5. **动效**：数字滚动 count-up、图表进入动画、列表自动上滚、地图散点呼吸/飞线、时钟每秒跳动；过渡平滑，不过度花哨。

## 三、页面结构与内容模块（按 1920×1080 布局）
- **顶部 Header（高约 88px）**：左侧实时日期/星期/时钟（每秒更新）；居中主标题「如意数据大屏」+ 副标题「RuyiBigScreen · Real-time Data View」；右侧状态徽标（系统在线 / 天气 / 刷新时间）。
- **左列（3 个面板）**：
  1. 核心指标卡：4 个 KPI（总访问量、在线用户、今日订单、转化率 %），带图标、单位与同比箭头，数字 count-up。
  2. 实时销量趋势：平滑面积折线图（多系列、渐变填充，24 小时或 7 日）。
  3. 品类销售占比：环形饼图（donut），中心显示总额。
- **中列（最大区域）**：
  1. 中国地图：省级着色 + 主要城市散点/气泡 + 2~3 条飞线（effectScatter + lines），悬停显示数值。
  2. 城市销量排行榜：Top10 横向条形榜，自动滚动循环。
- **右列（3 个面板）**：
  1. 渠道分布：横向条形图（APP / 小程序 / H5 / PC）。
  2. 设备与转化：两个仪表盘（gauge）或雷达图。
  3. 实时动态：滚动日志列表（如“用户 *** 在 ** 完成下单”），逐条新增并上滚。

## 四、数据策略
- 所有数据用 `const mock = {...}` 在前端定义，初始值合理、有业务含义。
- `setInterval` 每 2~3 秒轻微扰动数据（KPI 随机增减、折线追加新点、排行榜重排、日志插入新条），并调用 `chart.setOption(...)` 平滑更新；时钟每秒更新。
- 地图加载失败时捕获异常，该面板显示“地图资源加载失败”占位，不影响其他模块。
